# AI Eco Monitor - 이벤트-시그널 매핑 규칙 테이블 v1

## 설계 원칙

1. 각 측정변수에 대해 **관측 가능한 이벤트 유형**을 열거한다
2. 각 이벤트 유형에 대해 **시그널 방향**(Bullish/Bearish/Neutral)과 **강도 판정 기준**을 명시한다
3. 강도는 3단계: Strong(공시/정량), Moderate(신뢰 보도/정성), Weak(루머/추정)
4. Confidence는 소스 품질로 결정: High(공시/실적/공식발표), Medium(신뢰매체 보도), Low(업계 소문/추정)
5. 이 테이블은 LLM 프롬프트의 "판정 기준"으로 직접 활용된다
6. 시그널 전파 경로(Tier 1→2→3)에 따른 시간 지연(lag)을 명시한다

---

## 범례

- 방향: ▲ Bullish (메모리 수요 상승) / ▼ Bearish (수요 하락) / ◆ Structural (구조 변화, 방향 판단 보류)
- 영향 메모리: HBM / DRAM / NAND / LPDDR / 복합
- 시간축: 즉시(0-3M) / 단기(3-6M) / 중기(6-12M) / 장기(12M+)

---

## Tier 1: 연산 능력의 생산 — A (Physical Infra) + B (Semiconductor) + C (Cloud/Compute)

이 계층은 물리적 연산 능력을 생산하는 단계다. 전력/냉각/DC(A) + 반도체/장비/서버(B)가 결합되어 클라우드/컴퓨트(C)라는 연산 용량으로 출력된다. 메모리 수요에 가장 직접적으로 연결되며, 정량화 가능성이 높다.

### Tier별 역할과 메모리 수요 관계

### 프로젝트 목적: 메모리 수요 방향성 예측

세 Tier 모두 "메모리 수요"를 바라보되, 관점이 다르다. 세 관점의 곱이 실제 수요 전망이 된다.

```
실제 메모리 수요 = Tier 1 (인프라 총량) × Tier 2 (단위당 메모리 계수) × Tier 3 (가동률/실현율)
```

| Tier | 구성 | 수요 예측 관점 | 핵심 질문 | 시간축 |
|------|------|--------------|----------|--------|
| Tier 1 | A+B+C | 인프라 투자 규모 기반 수요 추정 | DC 용량, GPU/ASIC 출하, Hyperscaler/Neocloud capex를 감안할 때 물리적으로 메모리가 얼마나 필요한가? | 3-18M |
| Tier 2 | D→E→F | 모델/기술 트렌드 기반 수요 계수 변화 | 파라미터 스케일링 vs 효율화(양자화, 압축, offloading, 하네스)가 단위당 메모리 소비를 어떻게 바꾸는가? 대용량-고성능 트레이드오프에서 어떤 메모리 중심으로 진행되는가? | 6-12M |
| Tier 3 | G+H+J | 실제 활용 기반 수요 실현 검증 | AI 활용이 실제로 활발하게 일어나고 있는가? 어떤 앱/섹터가 추론 수요를 드라이브하며 성장 잠재력을 갖고 있는가? 투자가 정당화되는가? | 12-24M |
| 횡단 | I | 규제/보안에 의한 수요 구조 변형 | On-prem 전환, 보안 오버헤드, 규제에 의한 도입 지연/가속 등이 수요 구조를 어떻게 바꾸는가? | 가변적 |

### Tier 간 전파 규칙

**순방향 전파 (정상 시그널 흐름)**

| 조합 | 해석 | 메모리 수요 판단 |
|------|------|-----------------|
| T1 ▲ + T2 ▲ + T3 ▲ | 전 계층 동시 성장. 인프라 확장 + 모델 발전 + 수요 확인 | 강한 상승. 모든 메모리 제품 수혜 |
| T1 ▲ + T2 ▲ + T3 - | 인프라/모델은 성장하나 최종 수요 미확인 | 상승이나 주의. 과잉 투자 초기 징후 가능 |
| T1 ▲ + T2 - + T3 - | 인프라만 확장, 활용/수요 부재 | 경계. 공급 과잉 리스크 |
| T1 - + T2 ▲ + T3 ▲ | 수요/활용은 강하나 인프라 병목 | 단기 가격 상승. HBM 프리미엄 확대 가능 |
| T1 ▼ + T2 ▼ + T3 ▼ | 전 계층 위축 | 강한 하락. 사이클 하강 |

**역방향 전파 (피드백 루프)**

- T3 수요 확인 → T1 추가 투자 정당화 (12-24개월 lag)
- T2 모델 효율화 → T1 연산 수요 변화 (Jevons Paradox: 효율화가 오히려 총수요 증가)
- I 규제 강화 → T2 On-prem 전환 수요, T3 도입 지연

### Tier 2 내부 전파: D → E → F

Tier 2는 단순 병렬이 아닌 **직렬 체인**이므로 내부 전파 방향이 중요:

| 조합 | 해석 |
|------|------|
| D ▲ + E ▲ + F ▲ | 데이터 → 모델 → 도구 전 체인 성장. 추론 수요 가속 확인 |
| D ▲ + E ▲ + F - | 모델은 발전하나 도구/에이전트 생태계 미성숙. 수요 실현 지연 |
| D - + E ▲ + F ▲ | 데이터 없이 모델/도구 성장. 지속가능성 의문 (데이터 병목) |
| D ▲ + E - + F ▲ | 데이터/도구는 있으나 모델 정체. 기존 모델 기반 추론 수요만 유지 |

## 부록 2: 히트맵 시각화 스펙 (스케치 2페이지 기반)

### 노드별 시그널 집계 방식

각 대분류(A-J)의 노드 색상은 해당 대분류 내 모든 측정변수의 시그널을 가중 합산:

```
노드 시그널 = Σ (각 변수의 방향 × 강도 × Confidence 가중치)
```

- 방향: ▲ = +1, ▼ = -1, ◆ = 0
- 강도: Strong = 3, Moderate = 2, Weak = 1
- Confidence: High = 1.0, Medium = 0.7, Low = 0.4

### Tier 박스 시그널 (Tier 단위 집계)

```
Tier 시그널 = Σ (소속 노드 시그널) / 소속 노드 수
```

Tier 박스에 배경색을 적용하여 계층 전체의 방향을 즉시 파악 가능

### 색상 매핑

- 강한 Bullish (값 > +5): 진한 녹색 (#16a34a)
- Bullish (+2 약 +5): 연한 녹색 (#86efac)
- Neutral (-2 약 +2): 회색 (#9ca3af)
- Bearish (-5 약 -2): 연한 적색 (#fca5a5)
- 강한 Bearish (< -5): 진한 적색 (#dc2626)

### 화살표(전파 경로) 표현

Tier 간 화살표:
- 굵기: 전파되는 시그널 강도에 비례
- 색상: Bullish = 녹색, Bearish = 적색, Structural = 청색
- 펄스 애니메이션: 최근 7일 내 신규 시그널 발생 시 펄스 효과

Tier 2 내부 화살표 (D→E→F):
- 직렬 체인을 별도 표현하여 내부 전파 방향 시각화

I(Security/Governance) 화살표:
- Tier 2, Tier 3에만 연결 (Tier 1과는 화살표 없음)
- 점선으로 표현 (횡단 영향)

### 노드 클릭 시 드릴다운

각 노드(A-J) 클릭 → 해당 대분류의 측정변수별 시그널 상세 표시
- 각 변수의 최근 이벤트 리스트
- 방향/강도/Confidence 표시
- 근거 뉴스 원문 링크
